# Reports Hub 개선 로드맵

## 목표

레포트 목록을 많이 보여주는 서비스에서 벗어나, 사용자가 오늘 확인해야 할 변화와 근거를 빠르게 판단할 수 있는 리서치 도구로 발전시킨다.

## 1. 홈 `오늘의 핵심 변화`

### 사용자 가치

- 오늘 발간된 전체 리포트 흐름을 한 화면에서 파악한다.
- 목표가 상향, 투자의견 변경, 신규 커버리지처럼 중요한 변화부터 확인한다.

### 필요한 데이터

- 당일 리포트 수와 신규 등장 종목
- 목표가 및 투자의견의 이전 값과 현재 값
- 종목별 리포트 발간 증권사 수

### 구현 단계

1. 기존 검색 API 데이터로 발간량과 종목 수를 먼저 표시한다.
2. 목표가·투자의견 이력을 저장한 뒤 주요 변화 순위를 추가한다.
3. 사용자 키워드와 즐겨찾기를 반영한 개인화 순위를 제공한다.

### 완료 기준

- 홈 첫 화면에서 오늘의 주요 변화 3~5개를 확인할 수 있다.
- 각 항목에서 관련 리포트로 바로 이동할 수 있다.

## 2. `summary_id` 직접 진입과 종목별 연관 리포트

### 사용자 가치

- 홈에서 선택한 종목요약을 FnGuide 화면에서 즉시 연다.
- 현재 요약 아래에서 같은 종목의 다른 증권사 및 과거 리포트를 연속해서 확인한다.

### 필요한 데이터

- `summary_id`, 표준 종목 코드
- 동일 종목의 요약 및 일반 리포트 연결 정보

### 구현 단계

1. `/fnguide?summary_id=...` 직접 진입과 자동 스크롤을 지원한다.
2. 선택 요약을 상단에 고정하고 나머지 목록을 아래에 배치한다.
3. 종목 코드로 일반 리포트와 FnGuide 요약을 함께 조회한다.

### 완료 기준

- 홈의 종목요약이 각각 정확한 요약으로 연결된다.
- 선택한 종목의 관련 리포트를 화면 이동 없이 이어서 볼 수 있다.

## 3. 목표가 변화와 상승여력

### 사용자 가치

- 목표가 숫자만 보는 대신 현재 기준 기대수익과 방향을 즉시 판단한다.
- 증권사별 목표가 차이를 비교하기 쉬워진다.

### 필요한 데이터

- 현재 목표가와 직전 종가
- 이전 목표가와 이전 투자의견

### 구현 단계

1. 목표가와 직전 종가로 상승여력을 계산한다.
2. 이전 목표가가 확보되면 상향·유지·하향 상태를 표시한다.
3. 종목별 증권사 목표가 분포와 중앙값을 제공한다.

### 완료 기준

- 유효한 가격 데이터가 있는 FnGuide 카드에 상승여력이 표시된다.
- 잘못된 값이나 0은 계산에서 제외된다.

## 4. 실제 알림 이력과 읽음 처리

### 사용자 가치

- 종 아이콘에서 어떤 키워드가 어떤 리포트와 일치했는지 확인한다.
- 읽지 않은 알림과 이미 확인한 알림을 구분한다.

### 필요한 데이터

- `notification_id`, `user_id`, `report_id`
- `matched_keyword`, `created_at`, `read_at`

### 구현 단계

1. 알림 생성 및 목록 API를 추가한다.
2. 읽음 처리와 미확인 개수 API를 추가한다.
3. 헤더 플로팅 알림에 실제 데이터를 연결한다.

### 완료 기준

- 종 아이콘에 실제 미확인 개수가 표시된다.
- 알림 클릭 시 해당 리포트가 열리고 읽음 상태가 저장된다.

## 5. 동일 종목·동일 이슈 리포트 묶기

### 사용자 가치

- 여러 증권사가 반복해서 다룬 동일 이슈를 하나의 흐름으로 파악한다.
- 시장 컨센서스와 이견을 개별 문서를 열기 전에 비교한다.

### 필요한 데이터

- 표준 종목 코드, 섹터, 핵심 키워드
- 발간 시각과 요약 임베딩 또는 이슈 분류 결과

### 구현 단계

1. 같은 종목·같은 날짜 리포트를 기본 그룹으로 묶는다.
2. 제목과 키워드 유사도로 이슈 단위를 분리한다.
3. 그룹별 공통 결론, 증권사 수, 목표가 범위를 요약한다.

### 완료 기준

- 같은 이슈의 리포트가 하나의 묶음으로 표시된다.
- 묶음에서 공통 의견과 반대 의견을 구분해 볼 수 있다.

## 권장 실행 순서

1. 목표가 상승여력
2. `summary_id` 직접 진입
3. 홈 `오늘의 핵심 변화`
4. 실제 알림 이력
5. 동일 이슈 묶기

상승여력은 기존 데이터만으로 구현할 수 있다. 이후 항목은 종목 코드 정규화와 백엔드 데이터 모델 보강이 선행되어야 한다.
