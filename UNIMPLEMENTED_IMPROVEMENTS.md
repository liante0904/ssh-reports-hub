# 미반영 개선사항

## 목적

현재 대화에서 제안되었지만 아직 구현되지 않았거나, 데이터 및 UX 결정이 더 필요한 작업을 관리한다.

## 낮은 위험도

| 항목 | 현재 상태 | 다음 작업 |
|---|---|---|
| FnGuide 필터 상태 URL 보존 | 날짜, 태그, 검색 조건이 새로고침 후 초기화됨 | `date`, `facet`, `q`, `provider` 쿼리 파라미터 설계 및 복원 |
| FnGuide 헤더 패널 접기 | 항상 노출 중 | 모바일 높이를 검토한 뒤 접기 버튼 또는 sticky 필터 적용 |
| 로그인 배지 옆 빠른 로그아웃 | 내 정보 플로팅 메뉴에는 로그아웃이 있음 | 중복 UI가 필요한지 결정 후 데스크톱에만 적용 |
| 즐겨찾기 접근 UI 정리 | 기존 기능은 있으나 헤더 접근 구조가 명확하지 않음 | 알림·계정 메뉴와 겹치지 않는 위치 결정 |

## 중간 위험도

| 항목 | 필요한 선행 작업 | 위험 요소 |
|---|---|---|
| FloatingNav 완전 제거 | 모바일·데스크톱별 실제 사용 경로 점검 | 테마 변경과 새로고침 접근 경로 변경 |
| 새로고침 버튼 제거 | iOS Safari, Chrome 및 설치형 PWA 동작 확인 | 데스크톱과 일부 웹뷰에는 당겨서 새로고침이 없음 |
| FnGuide 전용 리포트 리더 | 목록과 상세 화면의 책임 및 URL 구조 결정 | 뒤로가기, 필터 상태, 모바일 스크롤 복원 |
| 스크롤 위치에 따른 `summary_id` 자동 변경 | IntersectionObserver 기반 활성 카드 판정 | 스크롤 중 URL·히스토리가 과도하게 변경될 수 있음 |
| 오늘의 리서치 브리핑 | 실제 데이터 기반 선정 기준 정의 | 목업을 실제 투자 정보처럼 오인할 가능성 |

## 높은 위험도 또는 백엔드 필요

| 항목 | 필요한 데이터·API |
|---|---|
| 실제 리포트 알림과 읽음 처리 | 알림 목록, 미확인 개수, 읽음 처리 API 및 알림 저장소 |
| 꼭 봐야 할 리포트 자동 선정 | 중요도 점수, 중복 이슈, 사용자 관심 종목 및 선정 근거 |
| 직전 종가와 상승여력 완성 | 신뢰 가능한 종가 수집, 거래일 보정, 종목 코드 정규화 |
| 목표가·투자의견 변화 추적 | 이전 목표가와 의견 이력 테이블 |
| 상위 투자자용 분석 대시보드 | 컨센서스, 실적 추정 변화, 밸류에이션 및 시계열 데이터 |
| 동일 이슈 리포트 묶기 | 제목·본문 유사도, 표준 종목 코드, 이슈 분류 또는 임베딩 |

## 완료된 관련 기반

- VS Code Makefile 확장 추천 및 Makefile 인식 설정
- FnGuide `summary_id` 직접 진입과 자동 스크롤
- 선택 레포트 이전·다음 이동
- 최신 발간일 기본 조회
- 종목·증권사·작성자 태그 필터
- 같은 날짜의 동일 종목 리포트 묶음
- 재무 수치 강조와 카드 본문 가독성 개선
